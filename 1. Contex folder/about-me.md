# About Me (for Claude Code)

## 1. Context

I am Head of Treasury at a bank in Serbia, but I actively build tools and products.

I am NOT a professional developer, but:
- I understand logic, structure, and data
- I can follow code with explanation
- I want to learn by building

Most of what we build should be:
- Practical
- Simple to run
- Easy to modify later

---

## 2. What I Build

Typical projects:

### Data & Dashboards
- Banking sector dashboards (Power BI, Python)
- Liquidity and ALM tools
- Market data visualizations

### Financial Tools
- Investment product simulations
- Gold trading / pricing tools
- Scenario analysis (rates, FX, liquidity)

### Small Apps
- Internal tools (web apps, scripts)
- Simple client-facing tools
- Automation scripts

---

## 3. Coding Principles (VERY IMPORTANT)

### 1. Keep It Simple
- Avoid overengineering
- Prefer simple working solution over perfect architecture

### 2. Modular Code
- Split logic into clear functions
- Avoid long monolithic scripts

### 3. Readability > Cleverness
- Use clear variable names
- Add short comments where needed

### 4. Minimal Dependencies
- Avoid heavy libraries unless necessary
- Prefer standard libraries when possible

---

## 4. How I Want You to Write Code

### Always:
- Explain what the code does (before or after)
- Highlight where I can change parameters
- Keep setup simple

### Structure:
1. Short explanation
2. Full working code
3. Instructions to run it

### If complex:
- Break into steps
- Build incrementally

---

## 5. Preferred Stack

Use these by default unless I say otherwise:

### Data & Analysis
- Python (pandas, numpy)
- Excel integration if useful

### Visualization
- Plotly (preferred)
- Matplotlib (if simpler)

### Apps / UI
- Streamlit (default choice)
- Simple HTML/CSS/JS if needed

### Storage
- CSV / Excel first
- SQLite if needed

Avoid:
- Complex frameworks (unless really necessary)
- Overly abstract architectures

---

## 6. UI / UX Expectations

I care a lot about usability.

Always:
- Make interfaces clean and simple
- Think like an executive user
- Minimize clicks

Prefer:
- Filters at the top
- Clear charts
- Key metrics visible immediately

Ask yourself:
"Can this be understood in 5 seconds?"

---

## 7. How to Help Me Best

When I ask for something:

### DO:
- Suggest better structure if needed
- Improve my idea (don't just execute)
- Think one step ahead

### DO NOT:
- Overcomplicate
- Add unnecessary features
- Assume I want "enterprise-level architecture"

---

## 8. Debugging & Iteration

When something breaks:

1. Explain the issue simply
2. Show exact fix
3. Provide corrected code

If multiple options exist:
- Recommend ONE best option

---

## 9. Output Style

Use:
- Clear sections
- Bullet points
- Short explanations

For code:
- Clean formatting
- No unnecessary comments
- Focus on usability

---

Build things that:
- Work immediately
- Are easy to adjust
- Solve real problems

Not things that:
- Look impressive but are hard to use
- Require heavy setup
- Are difficult to maintain
